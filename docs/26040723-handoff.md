## 4. 关键发现

1. **WinUI3 UI 线程死锁**：在 `App()` 构造函数中同步等待异步方法会导致窗口永远不显示（MainWindowHandle=0），必须放在 `OnStart()` 中异步执行。
2. **`PropertyNameCaseInsensitive` 必须设为 true**：否则 PascalCase JSON key 无法被 camelCase policy 匹配。
3. **GLM Service 缺少 Auth header**：原实现用 `PostAsJsonAsync` 不带 header，必须手动加 `Authorization: Bearer`。
4. **HttpClient 60 秒超时不够**：OCR + GLM 分析链路可能需要 1-2 分钟。
5. **`SettingsViewModel` 是 Singleton**：构造函数只读一次 settings，所以加了 `ReloadFromSettings()` 方法在 `SettingsPage.OnAppearing` 中调用。

## 7. 风险与注意事项

- **不要把异步初始化放回 `App()` 构造函数**：会死锁 UI 线程，窗口不显示。
- **不要删 `PropertyNameCaseInsensitive = true`**：否则手写 PascalCase appsettings.json 会读空。
- **`appsettings.json` 每次构建不会被复制到输出目录**：如果执行了 `dotnet clean` 或全量重建，文件可能丢失，需要手动复制。
- **HttpClient 是 Singleton**：所有服务共享同一个实例，超时设置影响所有 HTTP 调用。
- **图片文件较大时 PaddleOCR 可能超时**：2-3MB 的图片 base64 后约 4MB，上传可能很慢。建议优先测试 185KB 的图片。
- **GLM-4.7 模型**：响应中包含 `reasoning_content` 字段（思维链），`GlmService` 目前只读 `content` 字段，这没问题。
