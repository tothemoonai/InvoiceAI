## 4. 关键发现

**配色系统**:
- `ThemeManager.cs` 从 XAML 资源字典读取颜色，提供便捷属性如 `ThemeManager.BrandPrimary`
- 资源未加载时有硬编码回退值，不会崩溃
- 颜色替换后 MainPage.cs 和 SettingsPage.cs 不再有硬编码颜色（除半透明遮罩 `#CC000000` 等特殊颜色）

**主题切换**:
- `SettingsPage.cs` 中有 `BuildThemeSelector()` 方法，三个 RadioButton 绑定到 `SettingsViewModel.IsAutoTheme/IsLightTheme/IsDarkTheme`
- `SettingsViewModel` 的 `ApplyTheme()` 方法调用 `Application.Current.UserAppTheme` 设置主题
- `App.cs` 的 `CreateWindow()` 调用 `ApplyThemeMode()` 读取 `AppSettings.ThemeMode`

**导入功能**:
- 拖拽导入已完全移除（`App.cs` 无 `SetupWindowsDragDrop`，`MainPage.cs` 无 `BuildDropZone`）
- 导入仅通过 `OnImportClicked` → `PickFilesAsync` → `_importVm.ProcessFilesCommand`
- `ImportViewModel.Cancel()` 使用 `CancellationTokenSource` 中断异步流程

**字体**:
- 所有字体样式定义使用 `OpenSansSemibold` / `OpenSansRegular` 回退
- 未来如需替换字体，只需修改 `Styles.xaml` 中的 `FontFamily` 值并下载对应 TTF 文件

## 7. 风险与注意事项

**容易误判的点**:
1. **导入覆盖层误触发** — 之前版本中，拖动窗口边框时会误触发 `DragEnter` 事件，导致覆盖层显示。已通过移除 `SetupWindowsDragDrop` 解决。如果后续重新添加拖拽导入，务必注意这一点。
2. **IsProcessing 绑定** — `_importOverlay.IsVisible` 之前自动绑定到 `_importVm.IsProcessing`，导致任何原因设置 `IsProcessing = true` 都会显示覆盖层。建议改为手动控制（点击导入按钮时显示，取消/完成时隐藏）。
3. **字体文件缺失** — `Styles.xaml` 中定义了 `PoppinsBold`、`NotoSansSCRegular` 等字体引用，但实际未下载对应 TTF 文件。MAUI 会回退到系统字体，不会崩溃，但视觉效果会打折扣。
4. **测试文件不同步** — `ImportViewModelTests.cs` 的 `CreateVm()` 方法可能需要更新构造函数参数。

**不建议继续的方向**:
- 窗口拖拽导入功能（用户明确表示不需要）
- 字体下载（网络限制导致无法下载，可后续手动添加）
