## 1. 项目概述

**InvoiceAI** 是一个 Windows 桌面优先的日本发票（適格請求書/インボイス）智能识别与管理工具。

**核心流程**: 用户导入图片/PDF → 百度 OCR 提取文本 → GLM API 智能整理（严格遵循日本国税庁适格请求书 6 项规则）→ SQLite 本地存储 → 分类查询 + Excel 导出。

**目标用户**: 在日本工作的中文/日语用户。

**设计文档**: `docs/superpowers/specs/2026-04-05-invoice-ai-design.md`
**实施计划**: `docs/superpowers/plans/2026-04-05-invoice-ai-implementation.md`

## 2. 技术栈

| 组件 | 技术 |
|---|---|
| 框架 | .NET MAUI 9 (net9.0-windows10.0.19041.0) |
| UI | C# Markup (CommunityToolkit.Maui.Markup) — 不使用 XAML |
| 架构 | MVVM + CommunityToolkit.Mvvm |
| OCR | 百度 OCR API (直接处理图片和 PDF，不需要额外 PDF 库) |
| AI | GLM API (api.bigmodel.cn)，模型 TBD，默认 glm-4-flash |
| 数据库 | EF Core + SQLite (Code First) |
| Excel | EPPlus 8.5+ |
| 双语 | .resx 资源文件 (中文默认 + 日语) |

**MAUI 9 API 注意** (必须遵守):
- 用 `CollectionView` 不用 `ListView` (已废弃)
- 用 `Border` 不用 `Frame` (已废弃)
- 用 `MainThread.BeginInvokeOnMainThread()` 不用 `Device.*` (已废弃)
- 用 `VerticalStackLayout`/`HorizontalStackLayout` 不用兼容 `StackLayout`
- EPPlus 8+: `ExcelPackage.LicenseContext` 带警告抑制

## 8. 关键设计决策

1. **PDF**: 百度 OCR 直接处理，不需要 PdfiumNet/PdfSharp
2. **GLM 模型**: 默认 glm-4-flash，可在设置中修改
3. **多模态**: 当前纯文本模式 (OCR 文本 → GLM)，未来可选增强
4. **分类存储**: appsettings.json (轻量配置数据)
5. **重复检测**: SHA256 FileHash
6. **数据库路径**: `%LOCALAPPDATA%/InvoiceAI/invoices.db`
7. **配置路径**: `%LOCALAPPDATA%/InvoiceAI/appsettings.json`
